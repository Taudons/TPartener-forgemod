# fairy-lantern-item

## 新增需求

### 需求：精灵提灯可用于生成伙伴实体
"精灵提灯"物品应允许玩家通过在有效位置点击右键来生成伙伴实体。

#### 场景：在有效位置使用物品
- **When** 玩家在有效方块（空气充足的方块）上使用精灵提灯右键点击时
- **Then** 新的 `TPartnerEntity` 应在目标位置生成
- **AND** 物品堆叠数量应减少 1

#### 场景：在无效位置使用物品
- **When** 玩家在无效位置（固体方块、空间不足）使用精灵提灯时
- **Then** 不应生成实体
- **AND** 物品堆叠数量不应减少

### 需求：精灵提灯具有正确的物品注册
精灵提灯应作为正确的 Forge 物品进行注册，并具有正确的物品属性。

#### 场景：物品已注册
- **When** 模组初始化时
- **Then** 精灵提灯物品应完成注册
- **AND** 物品应可通过 `ForgeRegistries.ITEMS` 访问

### 需求：精灵提灯具有显示名称
精灵提灯应具有可翻译的显示名称。

#### 场景：物品显示正确名称
- **When** 物品显示在玩家物品栏中时
- **Then** 名称应为 "item.tpartnermod.fairy_lantern"（或等效的翻译键）

### 需求：精灵提灯以正确的方向生成实体
实体生成时应面向使用该物品的玩家。

#### 场景：实体面向玩家
- **When** 玩家使用精灵提灯生成实体时
- **Then** 生成的实体应面向玩家
- **AND** 实体的偏航旋转（yaw）应与玩家的视线方向匹配
